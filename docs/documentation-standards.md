# Documentation Standards

All documentation is in /doc EXCEPT /readme.md which is a high level overview of the project.

Our projects maintain specific documentation to support the SDLC process.

## Required Documentation

### Planning Phase
- Project Charter
- Vision Document
- High-level Roadmap

### Requirements Phase
- Business Requirements Document
- User Stories
- Acceptance Criteria
  - Acceptance Tests use Gherkin
- Domain Glossary (Ubiquitous Language)

### Design Phase
- Architecture Overview
- Bounded Context Map
- Domain Model Diagrams
- Event Storming Results
- Event Model Documentation
- Technical Design Specification

### Development Phase
- Code Documentation
- API Documentation
- Implementation Notes

### Testing Phase
- Test Plan
- Test Cases
- Test Results

### Deployment Phase
- Deployment Guide
- Configuration Documentation
- Rollback Procedures

### Maintenance Phase
- System Documentation
- Troubleshooting Guide
- Change Log

## Documentation Format
Documentation should be written in Markdown and stored in version control.
